static修饰局部变量

⼀般情况下,对于局部变量在程序中是存放在栈区的,并且其⽣命周期在包含语句块执⾏结束时便结束了。但是如果⽤static关键字修饰的话,该变量便会存放在静态数据区,其⽣命周期会⼀直延续到整个程序执⾏结束。

但是要注意的是,虽然⽤static对局部变量进⾏修饰之后,其⽣命周期以及存储空间发⽣了变化,但其作⽤域并没有改变,还是只能在原来的语句块中使用。

static修饰全局变量

对于⼀个全局变量,它既可以在本⽂件中被访问到,也可以在同⼀个⼯程中其它源⽂件被访问(添加 extern进⾏声明即可)。⽤static对全局变量进⾏修饰改变了其作⽤域范围,由原来的整个⼯程可⻅变成了本⽂件可⻅。

修饰普通函数

⽤static修饰函数,情况和修饰全局变量类似,也是改变了函数的作⽤域。

修饰类成员函数

类中的静态成员函数类相关的,而不是和类的具体对象相关的。即使没有具体对象,也能调用类的静态成员函数。

static成员函数主要目的是作为类作用域的全局函数。类的静态成员函数没有this指针,这导致static成员函数:

     * 不能在内部访问类的非静态成员变量,调用非静态成员函数
     * 不能被virtual修饰

static修饰类成员变量

static成员变量属于整个类所拥有,类的所有对象共享⼀份static类成员变量拷⻉。

在C++中,static静态成员变量不能在类的内部初始化。在类的内部只是声明,定义必须在类定义体的外部,通常在类的实现文件中初始化。static关键字只能用于类定义体内部的声明中,定义时不能出现static。 

     double Account::Rate=2.25;    //在类的外部定义
